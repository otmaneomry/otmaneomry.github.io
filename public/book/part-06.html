Part VI — The Architect

Part VI — The Architect

The nouns, before the story (plain prose — skip freely if you already run Spring)

Bounded context — a part of the domain with its own model and its own language; the natural unit to split along, if you ever split. Modular monolith — one deployable whose internal borders are real enough to cut later. Strangler fig — evolving a system by routing traffic at the edge to new code, growing it while the old starves. Outbox — write the outgoing event in the same database transaction as the data, then relay it afterward — so "saved" and "announced" can never disagree. Saga — a cross-service flow built from local transactions plus compensating actions, because distributed transactions across services are not a real option. Virtual threads — cheap JVM threads that release their OS carrier while blocked: thread-per-request semantics without the pool ceiling. Reactive (WebFlux) — event-loop concurrency with explicit pipelines: the highest ceilings, and a different programming model with its own discipline. Native image — compiling the application ahead-of-time into a binary: near-instant start and low memory, paid for at build time.
