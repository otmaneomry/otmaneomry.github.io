Part IV — The Data

Part IV — The Data

The nouns, before the story (plain prose — skip freely if you already run Spring)

Persistence context — the map of entities Hibernate watches during a transaction. It caches by identity, detects your changes by diffing snapshots (dirty checking), and writes them at commit. Lazy loading — a fetch strategy: an association is loaded on first access rather than up front. N+1 — a query pattern: one query for a list, plus one more per element as each is touched. Transaction — the all-or-nothing unit of work, opened and committed by a proxy around an annotated method. Propagation — how nested calls share (or suspend) it. Isolation — what concurrent transactions are allowed to see of each other. Optimistic vs pessimistic locking — detect a conflicting write at commit via a version column, and retry or surface it — versus blocking other writers with a database lock while you hold the row. Repository — an interface Spring Data implements at runtime; method names and annotations define the queries. Projection — fetching exactly the columns a view needs into a record, instead of loading whole entities. Migration — versioned, append-only SQL scripts (Flyway) that evolve the schema. Connection pool — the small fixed set of database connections (Hikari) every request must borrow from.
