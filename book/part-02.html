Part II — The Container

Part II — The Container

The nouns, before the story (plain prose — skip freely if you already run Spring)

Scope — how many instances of a bean exist. The default is singleton: one shared instance for the whole application, used by every request at the same time. Bean lifecycle — the fixed sequence the container walks for every bean: construct → inject dependencies → run init callbacks → wrap in proxies if needed → serve → destroy at shutdown. BeanPostProcessor — container machinery that may inspect or replace each bean as it is built. Most "magic" annotations are implemented by one. AOP proxy — a generated wrapper that runs logic around your method (open a transaction, check a permission). Calls reach your method through the wrapper. Auto-configuration — Boot's conditional defaults: pre-written configuration that activates only when your classpath and properties say it applies. Starter — a dependency bundle that pulls in one feature's libraries at tested, compatible versions. Profile / configuration properties — named environment overlays (dev, prod) and typed objects bound from YAML or environment variables.
